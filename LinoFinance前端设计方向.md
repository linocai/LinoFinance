# LinoFinance 前端设计方向

版本：V0.1
日期：2026-05-16
配套文档：`LinoFinance前置计划.md`
目标平台：iOS 18+ / iPadOS 18+ / macOS 15+
默认展示币种：CNY
设计基线：Apple HIG 2025 / SwiftUI + Liquid Glass / SF Symbols 6
文档用途：作为前端施工的总体方向、信息架构、视觉与交互规范、关键页面骨架、组件复用、AI 与通知设计，并给出 MVP 落地顺序。

---

## 0. 总体设计哲学

LinoFinance 不是流水账 APP，它是一个**个人财务控制台 + AI 财务管家**。前端必须同时承载两种相反的体验诉求：

- **快**：iOS 一秒内能开始一句话记账。
- **慢**：macOS 上能放下表格、报表、对账、批量编辑。

由此引出 5 条贯穿全端的设计原则：

1. **结构清晰 > 视觉炫技**：金融数据本身是主角，UI 应让数字第一眼清楚。
2. **状态可见 > 操作步骤**：草稿 / 已确认 / 已发生 / 已报销 / 已结清，要在卡片层面就能看出来。
3. **可逆 > 阻断**：任何 AI 自动动作都可一键回滚；高风险动作必须走「计划 → 确认 → 执行」的三段式。
4. **币种永不丢失**：原币种和人民币折算永远成对出现，绝不只展示折算金额。
5. **iOS 与 macOS 共享一套语义，不共享一套布局**：组件、配色、术语、状态全端一致；导航与密度按平台特性各自最优。

## 1. 设计语言 Design Language

### 1.1 命名与品牌

- 产品名：LinoFinance
- 短名 / 图标文字：Lino
- 文案语气：克制、确定、像账房先生，不要营销腔。
  - 好：「6 月 5 日还 Chase $800」
  - 差：「快来还款，省下利息小妙招！」

### 1.2 配色（语义优先）

颜色全部走**语义令牌**，不直接用色值，方便深浅色和高对比度自适应。

| 令牌 | 用途 | Light | Dark |
|---|---|---|---|
| `surface/base` | 主背景 | System Background | System Background |
| `surface/raised` | 卡片背景 | Secondary System Bg | Tertiary System Bg |
| `surface/glass` | 浮层 / 悬浮按钮 | Liquid Glass Regular | Liquid Glass Regular |
| `text/primary` | 主文本 | Label | Label |
| `text/secondary` | 次文本 / 说明 | Secondary Label | Secondary Label |
| `accent/brand` | 品牌强调 | 深青蓝 `#1E6FD9` | 浅青蓝 `#5DA8FF` |
| `accent/cny` | 人民币标识 | `#C8102E` 偏深 | `#FF6B7A` |
| `accent/usd` | 美元标识 | `#2F855A` 偏深 | `#4ADE80` |
| `state/income` | 进账 | System Green | System Green |
| `state/expense` | 支出 | System Red | System Red |
| `state/credit` | 信用负债 | System Orange | System Orange |
| `state/pending` | 预计 / 草稿 | System Gray | System Gray |
| `state/warning` | 现金流压力 / 异常 | System Yellow | System Yellow |
| `state/ai` | AI 提示 / 建议 | System Purple | System Purple |

**重要约束**：

- 不用红绿表达「好 / 坏」，红绿只表达「资金方向（进 / 出）」。
- 信用负债不要用红色，避免和正常支出混淆；用橙色单独一类。
- 报销相关信息全程带紫色（state/ai）或品牌色边框，不让用户混进「真支出」里。

### 1.3 字体

- 文本：SF Pro / SF Pro Text。
- 数字：**SF Pro 等宽数字**（`.monospacedDigit()`），所有金额、日期、表格列全部启用等宽，列对齐永不抖动。
- 大金额标题（总览净资产）：SF Pro Rounded，更友好但仍财务感。
- 字号走 Dynamic Type，主要档位：
  - LargeTitle 34 / Title1 28 / Title2 22 / Headline 17 / Body 17 / Subheadline 15 / Footnote 13 / Caption 12

### 1.4 图标

- 全量使用 SF Symbols 6（含 multicolor、variable color）。
- 一级模块图标：
  - 总览 `chart.pie.fill`
  - 账户 `creditcard.fill` / `wallet.pass.fill`
  - 记账 `square.and.pencil`
  - 现金流 `arrow.left.arrow.right.circle.fill`
  - 报销 `arrow.uturn.left.circle.fill`
  - 信用 `creditcard.trianglebadge.exclamationmark`
  - 分析 `chart.line.uptrend.xyaxis`
  - AI `sparkles`
  - 通知 `bell.badge.fill`

### 1.5 形状与材质

- 圆角：卡片 16，按钮 12，输入框 10，浮层 22。
- 材质：iOS 大量使用 Liquid Glass（顶部栏、悬浮按钮、tab bar、bottom sheet handle 区）；macOS 用 `.sidebar`、`.regular` material 区分层级。
- 阴影：默认无阴影，靠材质和分层颜色拉开层级；只有「悬浮录入按钮」「AI 浮窗」用轻投影。

### 1.6 数字与币种展示规范

强制规则（前端组件层做约束，避免后期返工）：

1. **原币 + 折算成对**：例如 `$5,100 · 约 ¥36,720`，折算金额永远带「约」字。
2. **币种符号永远贴金额**：`¥3,200`、`$5,100`，不要 `3200 CNY`。
3. **负数用括号或前缀**：信用负债显示 `−¥6,048`，不要红字直接出现；颜色靠语义令牌。
4. **千分位**：永远使用 locale 化的千分位。
5. **粒度提示**：日汇总记录在金额前标 `≈` 或 tag「汇总」，表明非逐笔。
6. **状态前缀**：草稿 `· 草稿`、预计 `· 预计`、已发生不加前缀。

### 1.7 动效

- 进入与退出：标准 `.spring(response: 0.45, dampingFraction: 0.85)`。
- 金额变化（账户余额、净资产）：数字滚动 600ms。
- AI 思考态：变色 sparkles + 进度条；动作执行成功用一次轻触 haptic + 短闪光。
- 不做大幅 parallax 与花哨过场，**财务 APP 不应该让人觉得在玩**。

## 2. 跨端信息架构 IA

两端共享 9 大模块，但承载形态不同。

| 模块 | iOS 入口 | macOS 入口 |
|---|---|---|
| 总览 Dashboard | Tab 1 | 侧边栏 Top |
| 账户 Accounts | Tab 4「资产」内一级 | 侧边栏 |
| 记账 Entries | Tab 2 + 全局 + 按钮 + AI | 侧边栏 + 工具栏「新建」+ ⌘N |
| 未来现金流 Cash Flow | Tab 3 | 侧边栏 |
| 报销 Reimbursement | Tab 4「资产」内一级 | 侧边栏 |
| 信用 Credit | 总览 + 账户内嵌 | 侧边栏 |
| 分析 Reports | 总览顶部「分析」+ 设置内 | 侧边栏（重点模块） |
| AI 财务管家 | Tab 中央悬浮 + 全局快捷 | 工具栏 + ⌘K + 菜单栏图标 |
| 通知 Notifications | 系统通知 + Inbox（设置内） | 系统通知 + 侧边栏底部 Inbox |

### 2.1 iOS 主导航：5 Tab

```
[ 总览 ]  [ 记账 ]  [ 〈AI〉]  [ 现金流 ]  [ 资产 ]
```

- 中间 Tab 不是页面，而是**AI 录入悬浮按钮**：单击进入 AI 录入 Sheet，长按进入语音录入；双击进入完整 AI 对话页。
- 「资产」内含账户列表 + 报销中心 + 信用中心三个分段。
- 顶部右上角统一放：币种切换、时间范围、`⋯` 更多。

### 2.2 macOS 主导航：三栏 NavigationSplitView

```
┌─────────────┬─────────────────────┬───────────────┐
│ Sidebar     │ Content List        │ Inspector     │
│             │                     │               │
│ 总览        │ 记录 / 账户 / 现金流 │ 详情 / 编辑    │
│ 账户        │ 列表 + 筛选 + 批量  │ + AI 计划面板  │
│ 记账        │                     │               │
│ 现金流      │                     │               │
│ 报销        │                     │               │
│ 信用        │                     │               │
│ 分析        │                     │               │
│ ─────       │                     │               │
│ 通知 Inbox  │                     │               │
│ 设置        │                     │               │
└─────────────┴─────────────────────┴───────────────┘
```

- 工具栏永驻：`+ 新建`、`AI 对话`、`币种切换`、`时间范围`、`搜索 ⌘F`、`核对账户`。
- 菜单栏图标（Menu Bar Extra）：一键唤起 AI 输入框（类似 Raycast），不切到主窗口也能记账。
- Inspector：选中任何对象（记录 / 现金流 / 报销 / 分期）后右栏即变成详情，AI 提议改动时这里出现「待确认计划」。

## 3. iOS 前端设计

### 3.1 总览 Dashboard（Tab 1）

页面骨架（自上而下）：

1. **顶栏**：左侧头像/打招呼；右侧币种切换 + 通知铃铛。
2. **净资产大卡**（Hero）：
   - 上排：净资产金额（大字、SF Rounded、等宽数字，CNY 折算）。
   - 副排：余额合计 / 信用负债 / 美元资产折算 三个小数字横排。
   - 右上角小图：30 天趋势 mini chart（不带坐标轴）。
   - 长按：切换显示「不显示金额」（隐私模式）。
3. **未来 30 天现金流卡**：
   - 上：进账 / 出账 / 净额 三列。
   - 下：水平时间轴 7/30/90 切换，标出大事件（还款日、报销到账、订阅扣款）。
4. **待办胶囊条**（横滑）：
   - 待报销 X 笔、待还款 Y 张卡、订阅 N 即将扣款、AI 异常 M 条。每个胶囊点击进入对应中心。
5. **本月支出卡**（双视角切换）：
   - 顶部分段：报销前支出 / 个人净支出。
   - 中部环形分类图（最大 6 类，超出归「其他」）。
   - 右下角小标：「本月报销回款 ¥X，待回款 ¥Y」。
6. **AI 财务简报卡**：
   - 一段最多 3 行的人话总结（由 AI 生成，可刷新）。
   - 下方 2-3 个建议按钮（如「确认这笔大额支出？」「为 iCloud 创建订阅？」）。
7. **底部留 padding**，避免被悬浮 AI 按钮遮挡。

### 3.2 记账 Entries（Tab 2）

- 顶部分段控制：单笔 / 日汇总 / 多日汇总 / 草稿 / 全部。
- 默认时间维度：本周；可滑动顶部日期条切换日 / 周 / 月。
- 列表行（高密度）：
  ```
  ┌────────────────────────────────────────────┐
  │ 餐饮 · 日常吃饭                              │
  │ ¥120  · 微信钱包          5/16 12:30      │
  │ [可报销] [草稿]                              │
  └────────────────────────────────────────────┘
  ```
  - 左滑：作废 / 归档。
  - 右滑：标记可报销 / 切换分类。
  - 点击：进入详情 + 编辑。
- 顶部固定胶囊「+ 快速记账」与右下角 + 悬浮按钮两套入口（满足惯用左手与单手操作）。

#### 3.2.1 快速记账 Sheet（核心交互）

- 半屏 Sheet（可拖到全屏），顶部三种 tab：**AI 一句话** / **表单** / **粘贴**。
- AI 一句话：
  - 大文本输入框 + 麦克风按钮 + 拍照按钮。
  - 输入完成后下方实时出现「AI 解析结果卡」：分类明细、账户变动、是否报销，全部可点开编辑。
  - 底部「确认 / 保存为草稿 / 取消」三按钮。
- 表单：分类、金额、币种、账户、日期范围、备注、附件（多附件）、可报销勾选。
- 粘贴：粘贴邮件 / 短信 / 微信对话，AI 自动抽取。

### 3.3 现金流 Cash Flow（Tab 3）

- 顶部时间切换：**7 天 / 30 天 / 90 天 / 自定义**。
- 上半屏：时间轴可视化（按日聚合，进出账双向条形 + 关键事件标记）。
- 下半屏：列表分组（今日 / 明天 / 本周 / 后续）：
  - 进账绿色左标、出账橙红色左标、还款蓝色左标、报销紫色左标。
  - 行右滑：标记已发生 / 取消 / 修改日期。
- 顶部加号：新建一次性 / 周期性现金流，模板含「订阅、还款、工资、房租、报销」。
- 底部悬浮卡：**未来 30 天净额**，常驻提示压力。

### 3.4 资产 Tab（Tab 4，三段式）

顶部 Segmented：**账户 / 报销 / 信用**。

#### 3.4.1 账户

- 上方两块：人民币资产合计 / 美元资产合计（带折算）。
- 列表分组：余额账户、信用账户、已归档。
- 行右侧显示原币种余额 + CNY 折算（信用账户显示应还）。
- 点击进入：账户详情页 → 余额变化曲线 + 变动列表 + 「核对」按钮。
- 信用账户额外显示：账单日、还款日、当前账单、未出账、分期余额（点击进分期计划）。

#### 3.4.2 报销中心

- 顶部聚合卡：当前垫付总额、预计回款、最长垫付天数。
- 列表按状态分组（可报销、已提交、已批准、已到账、被拒/放弃）。
- 行内显示：原币种金额、预计到账日、距今天数、原始消费链接。
- 长按行：批量提交 / 批量打印发票。

#### 3.4.3 信用中心

- 顶部聚合：信用负债总额、下次还款日、最近 90 天还款压力 mini chart。
- 卡片网格：每张信用卡一张大卡（封面色 + Logo + 应还 + 还款日倒计时）。
- 卡详情：账单、最低 / 全额、分期、还款历史。

### 3.5 AI（中央悬浮 Tab）

- 单击：弹出 **AI 录入 Sheet**（详见 3.2.1）。
- 长按：直接开始**语音录入**（带 Live 波形与转写）。
- 双击 / 在 AI Sheet 顶部「打开完整对话」：进入**AI 对话页**：
  - 标准 chat 形态，但每条 AI 消息下方紧跟「**结构化动作卡**」（详见 §6）。
  - 顶部历史下拉、底部输入框 + 麦克风 + 附件。
  - 右上角「AI 任务清单」按钮：列出未确认计划、最近回滚操作、AI 提议规则。

### 3.6 其他 iOS 关键体验

- **Widget 三档**：
  - Small：本月净支出 / 未来 7 天净额。
  - Medium：今天 + 明天现金流事件。
  - Large：净资产 + 未来 30 天图。
- **Lock Screen Widget**：「下次还款日 X 天 ¥Y」。
- **Live Activity**：报销提交后到账前持续展示状态；信用卡还款日临近时 inline 倒计时。
- **Shortcuts**：「记一笔」「问 AI 我这周还能花多少」「确认某项现金流已发生」。
- **App Intents**：所有结构化动作（CreateEntry、CreateCashFlowItem 等）全部暴露为 Intent，给 Siri / Spotlight / 快捷指令使用。
- **拍照发票**：进入相机后 VisionKit 自动文字识别，AI 补全分类、金额、账户。
- **隐私模式**：长按净资产卡或在控制中心一键模糊金额（用于地铁公开场合）。

## 4. macOS 前端设计

### 4.1 总览 Dashboard

- 顶部一行四块大数字：净资产 / 余额合计 / 信用负债 / 未来 30 天净额。
- 第二行两栏：左侧未来现金流热力时间轴，右侧 AI 财务简报 + 建议清单。
- 第三行：本月分类支出报表（带「报销前 / 报销后」切换、币种切换、时间维度切换）。
- 第四行：账户健康度（每个账户最近 30 天最大波动 + 最近核对时间，超过 14 天未核对标红）。
- 右栏 Inspector：默认关闭；点击任意卡片打开右栏对应详情。

### 4.2 记账 Entries

三栏布局威力体现：

- 中间：列表，可切换密度（紧凑 / 标准 / 舒适），支持多选、列排序、按账户 / 分类 / 日期分组。
- 顶部工具栏筛选：日期、账户、分类、币种、状态、是否报销、AI 创建。
- 右栏：选中后展示详情 + 即时编辑。多选时变成「批量操作面板」（批量改分类、批量标记报销、批量作废、批量转草稿确认）。
- 顶部「+ 新建」打开**全局记账面板**（Sheet）：和 iOS 完全一致的三 tab（AI / 表单 / 粘贴）。
- ⌘N 直接新建；⌘⇧A 唤起 AI 输入；⌘F 全局搜索。

### 4.3 现金流 Cash Flow

- 中栏：日历视图（月）+ 列表视图（按日）双 mode 切换。
- 日历格内显示当日净额 + 事件圆点（多色，按类型）。
- 右栏 Inspector：选中某日 → 列出当日所有现金流事件、可拖拽改日期；选中某事件 → 详情 + 状态切换 + 取消 / 已发生。
- 顶部 KPI 条：未来 7/30/90 天净额，悬停看明细。

### 4.4 报销中心

- 中栏看板 mode（Kanban）：列分别是「可报销 / 已提交 / 已批准 / 已到账 / 被拒-放弃」，卡片可拖动改状态。
- 切换为「列表 mode」时可批量导出发票、批量提交、批量标记到账。
- 右栏：选中报销 → 显示原始消费链接、附件预览（PDF/图片）、未来现金流状态、payer 联系备注。

### 4.5 信用中心

- 中栏：信用账户卡片墙 + 下方「未来 90 天还款压力」时间线。
- 选中卡 → 右栏出现账单详情、分期计划列表、每期到期日、还款历史。
- 顶部一键「核对账单」：粘贴账单文本 → AI 解析 → 比对系统记录 → 列出差异让用户逐项接受。

### 4.6 分析 Reports（macOS 重点页）

- 上方报表选择器（左 sidebar 内嵌）：本月总览 / 分类 / 报销前后 / 现金流压力 / 信用负债 / 订阅。
- 中栏：交互式图表（Swift Charts），支持 hover 看数值、点击下钻到记录。
- 右栏：当前报表的洞察自然语言（AI 给出 3-5 条），并可一键「保存为月度备忘」。
- 顶部：导出（CSV / JSON / PDF 月报），打印。

### 4.7 macOS 关键体验

- **菜单栏 Menu Bar Extra**：一键唤起小输入框（类似命令面板），输入「打车 38 微信」回车即记账并 toast「已保存草稿」。
- **快捷键**：
  - ⌘N 新建记录、⌘⇧N 新建现金流、⌘⇧A AI 对话、⌘K 命令面板、⌘F 搜索、⌘1-7 切模块、⌘⌥E 导出、⌘. 取消正在进行的 AI 任务。
- **多窗口**：记录列表、报表、AI 对话各可独立窗口（学 Mail / Finder）。
- **拖拽**：发票 PDF / 图片可直接拖到对应记录或报销卡。
- **打印 / PDF 导出**：月度报告专用模板。
- **Touch ID / 第二窗口锁**：打开 APP 或显示金额前可要求 Touch ID（设置项）。

## 6. AI 财务管家的前端设计

AI 是这款 APP 的灵魂。前端必须把「黑箱 LLM」翻译成「可见、可改、可逆」的财务动作。

### 6.1 三种 AI 入口

| 入口 | iOS | macOS |
|---|---|---|
| **快速一句话** | Tab 中央 + 单击 → 半屏 Sheet | 菜单栏图标 / ⌘⇧A 弹出小窗 |
| **完整对话** | AI Sheet → 「打开完整对话」/ Tab 双击 | 工具栏 AI 按钮 / 独立窗口 |
| **页面内提议** | 总览简报卡、列表上方横条、异常徽标 | 总览简报、Inspector 内提议面板 |

### 6.2 AI 对话页组成

每条 AI 回复由两部分构成：

- **自然语言段**：解释它的理解、提问、提醒。
- **结构化动作卡 `AIPlanCard`**：每个卡片对应一个可执行结构化动作（CreateEntry / CreateCashFlowItem / MarkReimbursable …）。

`AIPlanCard` 必须包含：

```
┌──────────────────────────────────────────────────┐
│ [动作类型 icon] 创建多日汇总记录                   │
│                                                  │
│ 日期范围：5/15 - 5/16                            │
│ 分类明细：餐饮 ¥320 / 娱乐 ¥180 / 停车 ¥40        │
│ 账户变动：微信 −¥300 / 招行信用卡 +¥240 应还       │
│ 状态：待确认                                      │
│                                                  │
│ [ 修改 ]   [ 保存为草稿 ]   [ 确认执行 ]           │
└──────────────────────────────────────────────────┘
```

- 「修改」展开为可编辑的字段表。
- 「确认执行」前对照该动作的**权限分级**决定是否要 Touch ID / 二次确认 / 仅按钮。
- 卡片左上角小色条标识动作风险：绿（低）/ 黄（中）/ 红（高）。

### 6.3 权限分级在前端的体现

| 风险 | 前端表现 |
|---|---|
| 低风险 | 卡片浅灰边、直接「确认」按钮即可。 |
| 中风险 | 卡片黄边、按钮变「确认执行」+ 一次轻 modal「将执行 X 条变动，是否继续？」 |
| 高风险 | 卡片红边、必须长按 1 秒或 Touch ID；底部固定「自动归档审计日志」提示。 |

### 6.4 AI 任务中心

独立小页面（iOS 设置内、macOS 侧边栏底部）：

- **待确认计划**：AI 创建但未确认的动作（最多保留 7 天，过期归档为草稿）。
- **最近执行**：可一键回滚（凡是结构化动作都可逆）。
- **AI 建议的规则**：例如「将所有 'Starbucks' 自动归类餐饮 → 咖啡」「Netflix 每月 25 日扣款 → 订阅」，需手动开启。
- **AI 异常提醒**：例如「本周餐饮高于过去 8 周均值 280%」。

### 6.5 AI 失败与不确定的表达

- 不要让 AI 静默失败。
- 「AI 不确定金额时」必须显示空位 + 红色叹号，让用户填补，绝不猜。
- 「AI 解析不到账户时」要列出可能账户并提问，不要默认任选。
- 对话中区分：**事实陈述**（普通气泡）、**问题**（带 ❓ 前缀）、**建议**（带 ✨ 前缀）、**警告**（带 ⚠ 前缀）。

## 7. 通知系统前端设计

### 7.1 通知样式

iOS / macOS 一致使用三层结构：标题 / 副标题 / 一段说明。所有通知必须带至少一个**直接动作**（不进入 APP 即可处理）。

| 通知 | 例子 | 直接动作 |
|---|---|---|
| 信用卡还款 | 「Chase 信用卡 3 天后还款 $800」 | 标记已还 / 推迟提醒 / 改金额 |
| 现金流到期 | 「明天预计收 ¥12,000 工资」 | 确认已发生 / 推迟 / 取消 |
| 报销 | 「机票 $500 报销已批准，预计 6/10 到账」 | 标记到账 / 查看 |
| 订阅 | 「iCloud 明天扣款 $9.99」 | 取消订阅提醒 / 改账户 |
| 分期 | 「电脑分期还剩 8 期，下期 5/30 $100」 | 查看计划 / 提前结清 |
| 账户核对 | 「14 天未核对微信余额」 | 立即核对 / 推迟 7 天 |
| 异常支出 | 「本周餐饮超均值 280%」 | 查看 / 忽略 |
| 现金流压力 | 「未来 30 天预计净支出 ¥8,000」 | 查看明细 / 忽略 |

### 7.2 通知 Inbox

- 系统中心之外，应用内提供 Inbox：可按类型筛选、批量已读、可静音某类。
- iOS 在设置中入口 + 顶部铃铛；macOS 在侧边栏底部独立项。

### 7.3 静音原则

- 用户可针对每条规则单独静音 / 调整提前天数。
- AI 创建的通知规则一定要标识「由 AI 创建」并可一键关闭。
- 不做营销类通知。

## 8. 状态：空 / 加载 / 错误 / 离线

| 状态 | 设计要点 |
|---|---|
| 空态 | 简短一句 + 可触达的下一步动作（通常引导到 AI 录入）。绝不只显示「暂无数据」。 |
| 加载态 | 局部 skeleton；总览净资产用「上一次值 + 灰色」直到刷新，避免数字跳成 0 引起焦虑。 |
| 错误态 | 明确原因 + 重试。涉及金额计算错误时不掩盖，标红 + 让用户看到差异。 |
| 离线 | 全部记账走本地优先；顶部条提示「离线模式，AI 回复将延迟」。所有结构化动作均可离线创建为草稿。 |
| 同步冲突 | 给出两个版本对比，让用户选择保留哪一版。审计日志全部记录。 |

## 9. 可访问性、隐私与本地化

### 9.1 可访问性

- 所有金额组件支持 VoiceOver：例如「招行信用卡，当前应还，人民币六千零四十八元」。
- Dynamic Type 全级支持，含 AX 系列大字号；列表行可自适应折行不截断。
- 高对比度模式：状态颜色保留语义同时提高对比。
- 减少动效模式：禁用数字滚动、净资产趋势动画。
- 完整键盘可访问（macOS）：Tab 顺序、Esc 关闭、Return 确认。

### 9.2 隐私

- 隐私模糊：总览 / Widget / 锁屏单独开关。
- 默认数据全本地（Core Data + CloudKit 可选同步）。
- 删除走软删除 + 审计日志；30 天后才真实清除。
- AI 调用：在设置中可选「使用云端 AI / 仅本地小模型」，并显示当前模式徽标。

### 9.3 本地化

- 文案 i18n key 全部基于 §2.3 术语表。
- 数字、货币、日期全部走 locale。
- 中英双语优先支持，后续可扩展。
- 货币展示规则不变（永远原币 + CNY 折算），不管 locale 如何。

## 10. 推荐技术栈与代码结构

### 10.1 技术栈

- **UI**：SwiftUI（iOS 18+ / macOS 15+），关键场景内嵌 UIKit / AppKit（如 NSOutlineView 替代复杂表格）。
- **状态管理**：Observation + `@Observable` ViewModel；跨视图共享走依赖注入容器。
- **本地存储**：SwiftData / Core Data（建议 SwiftData，因为模型清晰且支持 CloudKit）。
- **图表**：Swift Charts。
- **AI 调用**：抽象 `AIBroker` 协议，云端 / 本地两实现；产出结构化 `AIAction` 而不是自由文本。
- **通知**：UNUserNotificationCenter + App Intents + Live Activities。
- **同步**：CloudKit Private Database（V1）；后续可换 Sync Engine。
- **测试**：XCTest + Snapshot tests（针对 MoneyText、StatusTag 等关键组件）。

### 10.2 代码结构（建议）

```
LinoFinance/
├── App/                  # 入口、AppDelegate、Scene
├── Core/
│   ├── Models/           # SwiftData 模型
│   ├── Services/         # AccountService, EntryService, CashFlowService, AIBroker...

├── Domain/
│   ├── Actions/          # CreateEntry, CreateCashFlowItem, ... 全部结构化动作

├── DesignSystem/
│   ├── Tokens/           # 颜色 / 字体 / 间距令牌
│   └── Components/       # MoneyText, StatusTag, AIPlanCard ...
├── Features/
│   ├── Dashboard/
│   ├── Entries/
│   ├── CashFlow/
│   ├── Reimbursement/
│   ├── Credit/
│   ├── Accounts/
│   ├── Reports/
│   ├── AI/
│   └── Notifications/
├── Platform/
│   ├── iOS/              # iOS 专属：Tab、Widget、Live Activity、Shortcut
```

## 13. 验收 Checklist（前端层）

前端可被视为合格的最低标准：

- [ ] 所有金额组件统一走 `MoneyText`，原币 + CNY 折算永不丢失。
- [ ] 所有状态走 `StatusTag` 语义令牌；草稿 / 预计 / 已发生在列表层即可识别。
- [ ] 信用卡消费、还款、报销、分期在 UI 上互相不混淆（颜色、tag、入口分明）。
- [ ] 报表支持「报销前支出 / 报销回款 / 个人净支出」三视角切换。
- [ ] AI 提议必须以 `AIPlanCard` 形式出现，且按风险分级控制确认方式。
- [ ] 高风险动作有审计日志可视化（macOS Inspector 内可查 before/after）。
- [ ] iOS 提供 Widget、Live Activity、Shortcuts、App Intents 至少各 1 项。
- [ ] macOS 提供 Sidebar + Inspector、菜单栏快速记账、⌘K 命令面板、批量编辑。
- [ ] 全端通过 Dynamic Type、VoiceOver、Reduce Motion 三项基础可访问性测试。
- [ ] 离线可记账（草稿态），恢复连接后能与云端同步并显示冲突解决面板。
- [ ] 隐私模糊金额可一键切换，且对 Widget / 锁屏生效。

## 14. 待补与后续 Open Question

这些不阻塞 MVP，但在 M3 - M6 期间需要补齐：

- 信用卡卡面定制（用户能否上传 logo / 选颜色，还是按发卡行映射）。
- 报销中心是否支持「转交他人代办」流程（payer 不只是公司）。
- 多设备 AI 对话是否需要历史云端同步（涉及隐私设置）。
- macOS 是否需要「自带浏览器抓取账单 / 银行邮件解析」插件入口。
- iOS 是否提供 Standby Mode 专属视图（CarPlay 暂不考虑）。
- 是否需要「家庭共享」预备字段（V1 不做，但 model 可保留 ownerId）。

## 15. 一页总结

> **iOS = 一秒记账 + 一眼看清楚 + 一键确认**
> **macOS = 一屏全局 + 批量整理 + 深度分析**
> **AI = 全端共脑，但永远走结构化动作，永远可回滚。**
> **数字 = 永远原币 + 人民币折算，永远状态可见，永远可被审计。**

把上面四条做到位，这个 APP 就不会变成下一个「打开过两次就忘了」的记账软件，而是一台真正能托管个人现金流的财务控制台。
