# LinoFinance v1.2 Plan

> Feature release `v1.2.0` on top of the shipped `v1.1.7` baseline.
>
> Scope: **modernise the dual-platform login experience** (macOS + iOS)
> plus **light up iOS Push end-to-end** now that the paid Apple Developer
> Program unlocks the entitlements that were stripped in v1.1.0. The
> existing single-bearer-token model stays as an "admin escape hatch".
>
> Production currently runs `v1.1.7` (build `b16f8b3`, alembic
> `202605200001`, release `20260524-110200`). Local working branch is
> `release/v1.1.7` at `aad9bb9`. v1.2 cuts a fresh `release/v1.2.0` off
> `main` after `release/v1.1.7` is merged in P0.
>
> AI module work continues to be deferred — see §3.

---

## 0. Project Goal

Ship `v1.2.0` as a focused identity + push patch that:

1. introduces **Sign in with Apple** as the primary login path on both
   iOS and macOS clients,
2. adds a backend identity layer:
   - new tables `users` and `auth_sessions` (the **first** alembic
     migration since `202605200001`),
   - new endpoint `POST /api/v1/auth/apple` exchanging an Apple
     identity_token for a LinoFinance session token (still a Bearer
     token; format unchanged from the wire's perspective),
   - new endpoint `GET /api/v1/auth/me` for the client to verify the
     session token and learn the logged-in user,
   - new endpoint `POST /api/v1/auth/logout` to revoke the active
     session,
   - new endpoint `GET /api/v1/auth/sessions` and `DELETE
     /api/v1/auth/sessions/{id}` so the user can see and revoke
     remembered devices from Settings,
3. preserves the existing `LINOFINANCE_API_AUTH_TOKEN` env mechanism as
   an "admin token" that **bypasses** the new `users` table — i.e. the
   single-token model still works for ops curl / production scripts /
   the deploy smoke,
4. lights up **iOS APNs production** by:
   - re-adding the Push Notifications entitlement to the iOS app and
     widget (reversing the v1.1.0 commit `14bb6ee` strip),
   - adding `App Groups` and `Sign in with Apple` capabilities to the
     iOS bundles,
   - documenting a fresh paid-team profile flow,
   - shipping the `.p8` config docs to `/etc/linofinance/` (NOT git),
   - smoke-testing a real push to a real device from the production
     backend,
5. retires the v1.1.x "paste your token in Settings" UX in favour of a
   "Sign in with Apple" button — the legacy SecureField stays under a
   collapsed "高级设置 / 管理员 Token" disclosure so admins can still
   paste the env token when needed,
6. modernises Settings → 设备 to list `auth_sessions` rows for the
   current user with last-seen-at and a revoke action,
7. cleans up the v1.1-era `.planning/STATE.md` "Remaining" list,
8. bumps every visible version string to `1.2.0` (backend pyproject,
   `config.app_version`, all 12 Xcode `MARKETING_VERSION` /
   `CURRENT_PROJECT_VERSION` entries, `scripts/deploy-api.sh`
   `EXPECTED_VERSION` default), and produces a runnable `LinoF.app` at
   `/Applications/LinoF.app` with both `CFBundleShortVersionString =
   CFBundleVersion = 1.2.0`.

Out of scope:

- AI module work (still deferred; see §3),
- production live deploy (user-driven; the plan ends at producing the
  `.app` + an iOS Dev profile signed simulator/device build, NOT at
  pushing to `hz`),
- multi-user invitation / sharing flows — `users` table exists but the
  Apple-sign-in path always creates exactly one user the first time it
  sees a new `sub`. There is no admin UI to add second users in v1.2.
- iCloud Keychain session-token sync (recommend it as a v1.3 follow-up
  if cross-device drift starts feeling annoying; v1.2 each device signs
  in independently),
- Sign in with Apple **web** flow (no Service ID); the only Sign in
  with Apple Service ID is the **app** one, which uses the bundle ID
  directly,
- Username / password / email login. No "Forgot Password" UI exists.

## 2. Why Sign in with Apple (decision, not a question)

I evaluated the three candidates the user listed. **Recommendation: A —
Sign in with Apple as the primary login.** Here's why:

| | A — Sign in with Apple | B — Token + QR bridge | C — Username/password + SIWA |
|---|---|---|---|
| First-launch UX | Tap one button, Face ID, done | Open macOS, copy token, open iOS Settings, tap "scan QR" | Type email + password OR tap SIWA |
| Native to ecosystem | Yes, Apple feature parity with App Store | No, ad-hoc QR is one-off | Yes for SIWA, but custom for U/P |
| Maintenance | One JWT verification path; Apple owns the lifecycle | QR encoder + scanner + token reveal screen + screen-rotation logic | User table + password hashing + reset email + SIWA — duplicate code |
| Recovery if device lost | Sign in again on a new device with Apple ID, server links the same `sub` to the same `user_id` | Need to re-fetch the env token | Reset password email (requires SMTP, which we don't have) |
| Multi-user readiness | First Apple `sub` we see becomes the only user; v1.3 can add `+1 user` flow on top | Same | Same |
| Cost | Free, part of paid Dev Program | Free | SMTP / email-sending infra cost |

B is rejected because the UX is no better than what we have today.
C is overkill for a single-user personal-finance app.

A's only real downside is that an Apple-Sign-In implementation requires
trusting Apple's JWT issuer, which is fine: it's part of why we paid
$99/yr.

The plan therefore implements **A** with the admin token kept alive as
a deliberate escape hatch.

## P1 — Alembic Migration + SQLAlchemy Models (后端)

### Goal

Add two new tables and the matching SQLAlchemy models. **No data
backfill** is required — the existing single-token model continues to
work because the auth middleware (P3) tries the admin token first.

### New tables

#### `users`

| Column            | Type                        | Notes                                                          |
| ---               | ---                         | ---                                                            |
| `id`              | `VARCHAR(40)` PK            | KSUID-like (use existing `IDTimestampMixin` style; `uuid4().hex` is fine) |
| `apple_user_id`   | `VARCHAR(255)` UNIQUE       | Apple `sub` claim from the identity token. Indexed.            |
| `email`           | `VARCHAR(255)` NULL         | The email Apple returns *only on the very first sign in*. NULL after, never re-issued. |
| `email_verified`  | `BOOLEAN` NOT NULL DEFAULT FALSE | Apple's `email_verified` claim.                         |
| `display_name`    | `VARCHAR(120)` NULL         | Constructed from Apple's `name.firstName` + `lastName` only on first sign-in (the same way Apple returns them once).                        |
| `is_admin`        | `BOOLEAN` NOT NULL DEFAULT FALSE | Reserved for future. Always FALSE for users created via Apple sign-in; the env admin token bypasses the user table entirely. |
| `disabled`        | `BOOLEAN` NOT NULL DEFAULT FALSE | Set TRUE to lock a user out without deleting their data. |
| `created_at`      | `TIMESTAMPTZ` NOT NULL DEFAULT now() |                                                       |
| `updated_at`      | `TIMESTAMPTZ` NOT NULL DEFAULT now() ON UPDATE now() |                                       |

Unique constraint on `apple_user_id`.

#### `auth_sessions`

| Column          | Type                       | Notes                                                                  |
| ---             | ---                        | ---                                                                    |
| `id`            | `VARCHAR(40)` PK           |                                                                        |
| `user_id`       | `VARCHAR(40)` NOT NULL FK → `users.id` ON DELETE CASCADE |                                            |
| `token_hash`    | `VARCHAR(64)` NOT NULL UNIQUE | SHA-256 hex of the issued plaintext token. Plaintext NEVER stored. |
| `device_label`  | `VARCHAR(120)` NOT NULL    | Human-readable, e.g. "iPhone Air · iOS 18.4". The client sends it on sign-in. |
| `platform`      | `VARCHAR(16)` NOT NULL     | `ios` / `macos`.                                                       |
| `app_version`   | `VARCHAR(32)` NULL         | The version string the client reported at sign-in.                     |
| `issued_at`     | `TIMESTAMPTZ` NOT NULL DEFAULT now() |                                                              |
| `last_seen_at`  | `TIMESTAMPTZ` NOT NULL DEFAULT now() | Updated on every authed request via the middleware (P3).     |
| `expires_at`    | `TIMESTAMPTZ` NOT NULL     | Default = `issued_at + 365 days`. Configurable in `config.py` (`session_lifetime_days: int = 365`). |
| `revoked_at`    | `TIMESTAMPTZ` NULL         | NULL = active.                                                         |

Indexes:

- `auth_sessions_user_id_idx`
- `auth_sessions_token_hash_uq` (already implied by UNIQUE)
- `auth_sessions_active_idx` partial: `(user_id) WHERE revoked_at IS NULL` — speeds the "list active sessions for user" query.

signature against Apple's JWKS, and returns the validated claims.

### New file: `backend/app/services/apple_identity.py`

```python
from __future__ import annotations

import time
from dataclasses import dataclass
from typing import Optional

import httpx
from jose import jwt
from jose.utils import base64url_decode

APPLE_JWKS_URL = "https://appleid.apple.com/auth/keys"
APPLE_ISSUER = "https://appleid.apple.com"

_jwks_cache: dict | None = None
_jwks_cache_expires_at: float = 0.0
_JWKS_TTL_SECONDS = 3600


@dataclass(frozen=True)
class AppleIdentity:
    sub: str                  # Apple's stable user id
    email: Optional[str]
    email_verified: bool
    aud: str                  # bundle id, e.g. com.lino.linofinance.ios
    iss: str
    iat: int
    exp: int
    is_private_email: bool    # email ends with @privaterelay.appleid.com


class AppleIdentityError(Exception):
    pass


def _fetch_jwks() -> dict:
    global _jwks_cache, _jwks_cache_expires_at
    now = time.time()
    if _jwks_cache is not None and now < _jwks_cache_expires_at:
        return _jwks_cache
    response = httpx.get(APPLE_JWKS_URL, timeout=10.0)
    response.raise_for_status()
    _jwks_cache = response.json()
    _jwks_cache_expires_at = now + _JWKS_TTL_SECONDS
    return _jwks_cache


def verify_apple_identity_token(
    token: str,
    expected_audiences: set[str],
) -> AppleIdentity:
    """Verify an Apple identity_token against Apple's JWKS.

    `expected_audiences` is the set of bundle IDs the server will accept.
    For LinoFinance v1.2 this is exactly
    {"com.lino.linofinance.ios", "com.lino.linofinance"}.
    """
    try:
        unverified_header = jwt.get_unverified_header(token)
    except Exception as exc:
        raise AppleIdentityError(f"Invalid token header: {exc}") from exc

    kid = unverified_header.get("kid")
    alg = unverified_header.get("alg")
    if not kid or alg != "RS256":
        raise AppleIdentityError("Unsupported token header (kid/alg)")

    jwks = _fetch_jwks()
    key = next((k for k in jwks["keys"] if k["kid"] == kid), None)
    if key is None:
        # Cache may be stale — bust once.
        global _jwks_cache_expires_at
        _jwks_cache_expires_at = 0.0
        jwks = _fetch_jwks()
        key = next((k for k in jwks["keys"] if k["kid"] == kid), None)
    if key is None:
        raise AppleIdentityError("Token kid not in Apple JWKS")

    try:
        claims = jwt.decode(
            token, key,
            algorithms=["RS256"],
            audience=list(expected_audiences),
            issuer=APPLE_ISSUER,
        )
    except Exception as exc:
        raise AppleIdentityError(f"JWT verification failed: {exc}") from exc

    sub = claims.get("sub")
    if not sub:
        raise AppleIdentityError("Token has no sub claim")

    email = claims.get("email")
    return AppleIdentity(
        sub=sub,
        email=email,
        email_verified=str(claims.get("email_verified", "false")).lower() == "true",
        aud=claims["aud"],
        iss=claims["iss"],
        iat=int(claims["iat"]),
        exp=int(claims["exp"]),
        is_private_email=bool(email and email.endswith("@privaterelay.appleid.com")),
    )
```

## P3 — Auth Endpoints + Middleware Update (后端)

### Goal

Expose the new identity layer over HTTP and teach the middleware to
accept session tokens in addition to the admin env token.

### New endpoints

#### `POST /api/v1/auth/apple`

Request:

```json
{
  "identity_token": "<JWT from ASAuthorizationAppleIDCredential>",
  "authorization_code": "<optional, ignored in v1.2>",
  "device_label": "iPhone Air · iOS 18.4",
  "platform": "ios",
  "app_version": "1.2.0",
  "first_name": "Lino",
  "last_name": "Tsai"
}
```

`first_name` / `last_name` are only sent on the **first** sign-in for a
given Apple `sub` — Apple's behaviour. The server only saves them if
`users.display_name` is currently NULL.

Response 200:

```json
{
  "session_token": "<plaintext, 43-char url-safe base64>",
  "expires_at": "2027-05-27T03:14:15Z",
  "user": {
    "id": "<user_id>",
    "apple_user_id": "...",
    "email": "lino@privaterelay.appleid.com",
    "email_verified": true,
    "display_name": "Lino Tsai",
    "is_admin": false
  }
}
```

Errors:

- `400` — `{"detail": "Invalid Apple identity token: <reason>"}` if
  `AppleIdentityError`.
- `400` — `{"detail": "platform must be one of: ios, macos"}`.
- `403` — `{"detail": "User is disabled"}` if matching user exists with
  `disabled=true`.

Behaviour:

1. Validate `platform in {"ios", "macos"}`.
2. Validate `device_label` length 1..120.
3. `identity = verify_apple_identity_token(payload.identity_token,
   set(settings.apple_signin_audiences))`.
4. Upsert user by `apple_user_id`:
   - If new: insert with email/email_verified from Apple, display_name
     constructed from first/last name if provided. `is_admin=False`.
   - If existing: leave email/display_name alone (Apple only gives them
     once anyway). Update `email_verified` if Apple's claim flipped to
     True.
   - If `user.disabled`, raise 403.
5. Mint a new session token: `token = secrets.token_urlsafe(32)`,
   `token_hash = hashlib.sha256(token.encode()).hexdigest()`.
6. Insert `AuthSession` with `expires_at = now() +
   timedelta(days=settings.session_lifetime_days)`.
7. Return the plaintext token + user.

The plaintext token is the **only** time it leaves the server. Subsequent
requests use it via `Authorization: Bearer <token>`.

#### `GET /api/v1/auth/me`

Requires authed request. Returns:

```json
{
  "user": { ...same shape as above ... },
  "session": {
    "id": "<session_id>",
    "device_label": "...",
    "platform": "ios",
    "issued_at": "...",
    "last_seen_at": "...",
    "expires_at": "...",
    "app_version": "1.2.0"
  }
}
```

If the request is authed via the **admin env token** (not a session
token), return:

```json
{
  "user": null,
  "session": null,
  "admin": true
}
```

This lets clients distinguish "you're logged in as an Apple user" from
"you're using the env admin token". Used by the macOS Settings to
decide whether to render the session-list section.

#### `POST /api/v1/auth/logout`

Requires authed request via a session token (NOT the admin token; if
the admin token is used, return 400 `{"detail": "Admin token cannot
log out"}`). Marks the current session `revoked_at = now()`.
Returns 204.

#### `GET /api/v1/auth/sessions`

Requires session-token auth. Returns all active sessions for the user:

```json
{
  "items": [
    {
      "id": "...",
      "device_label": "iPhone Air · iOS 18.4",
      "platform": "ios",
      "app_version": "1.2.0",
      "issued_at": "...",
      "last_seen_at": "...",
      "expires_at": "...",
      "is_current": true
    },
    { ... }
  ]
}
```

`is_current` is True for the session whose token hash matches the one
the request authenticated with.

#### `DELETE /api/v1/auth/sessions/{session_id}`

Requires session-token auth and `session.user_id == requester.user_id`
(otherwise 404). Sets `revoked_at = now()`. Returns 204.

If `session_id` is the requester's *current* session, this is the same
as `/auth/logout` — allowed.

### Middleware change

`backend/app/core/middleware.py` `APIAuthMiddleware.dispatch` (line
80+) currently:

```python
expected_token = self.settings.api_auth_token or ""
actual_token = _extract_token(request)
if actual_token is None or not hmac.compare_digest(actual_token, expected_token):
    return 401
```

After v1.2:

```python
actual_token = _extract_token(request)
if actual_token is None:
    return _unauthorized()

# Admin escape hatch — env token still works for ops.
if self.settings.api_auth_token and hmac.compare_digest(actual_token, self.settings.api_auth_token):
    request.state.auth = AuthContext(mode="admin", user=None, session=None)
    return await call_next(request)

# Session token path.
session = _resolve_session_token(actual_token, db_factory=self.db_factory)
if session is None:
    return _unauthorized()

request.state.auth = AuthContext(mode="user", user=session.user, session=session)
# Touch last_seen_at out of band (cheap UPDATE without bumping updated_at).
_touch_session_last_seen(session.id)
return await call_next(request)
```

`AuthContext` is a small dataclass in `backend/app/core/auth_context.py`:

```python
@dataclass
class AuthContext:
    mode: str            # "admin" | "user"
    user: Optional[User]
    session: Optional[AuthSession]
```

`_resolve_session_token`:

1. Hash the candidate: `hashlib.sha256(candidate.encode()).hexdigest()`.
2. SELECT from `auth_sessions` WHERE `token_hash = :h AND revoked_at IS
   NULL AND expires_at > now()`.
3. If not found, return None.
4. Return the row (with `user` eagerloaded).

The middleware needs a DB session factory. Builder picks one of:

- Pass `SessionLocal` (the existing factory) at middleware
  construction; open + close a short-lived session inside `dispatch`.
- Or thread it via `request.state` if main.py already wires that. Look
  in `backend/app/main.py`.

`_touch_session_last_seen` is `UPDATE auth_sessions SET last_seen_at =
now() WHERE id = :id`. It's NOT in the request-blocking critical path;
fire-and-forget via a background task or just do it inline (a single
indexed update is < 1 ms). Builder uses inline for simplicity.

## 5. API Contract Diff vs v1.1.7

### New endpoints

```text
POST /api/v1/auth/apple
  body:
    identity_token        : string (required)
    authorization_code    : string (optional, ignored)
    device_label          : string (1..120, required)
    platform              : "ios" | "macos" (required)
    app_version           : string (<=32, optional)
    first_name            : string (optional)
    last_name             : string (optional)
  200:
    session_token         : string
    expires_at            : ISO-8601
    user                  : AuthUser
  400, 403 — see error table.

GET /api/v1/auth/me
  → 200 {user: AuthUser|null, session: AuthSession|null, admin?: bool}

POST /api/v1/auth/logout
  → 204 (session token only) | 400 (admin token)

GET /api/v1/auth/sessions
  → 200 {items: AuthSession[]} (each row has is_current bool)

DELETE /api/v1/auth/sessions/{session_id}
  → 204 | 404
```

### Modified behaviour

- Auth middleware now accepts session tokens (DB-backed) in addition to
  the env admin token. **Existing clients with an env admin token are
  unaffected.**
- `GET /api/v1/health` adds `"auth_modes": ["admin", "user"]` (was
  `["admin"]`) so a client can detect server-side support. Plan adds
  the field; it's free.

### Error codes table

| Endpoint | Code | Detail |
|---|---|---|
| `POST /auth/apple` | 400 | `"Invalid Apple identity token: <reason>"` |
| `POST /auth/apple` | 400 | `"platform must be one of: ios, macos"` |
| `POST /auth/apple` | 422 | pydantic validation (missing fields, length) |
| `POST /auth/apple` | 403 | `"User is disabled"` |
| `GET /auth/me` | 401 | `"Missing or invalid API token"` |
| `POST /auth/logout` | 400 | `"Admin token cannot log out"` |
| `POST /auth/logout` | 401 | `"Missing or invalid API token"` |
| `GET /auth/sessions` | 401 | as above |
| `GET /auth/sessions` | 400 | `"Admin token cannot list sessions"` |
| `DELETE /auth/sessions/{id}` | 404 | `"Session not found"` (or not owned by requester) |
| `DELETE /auth/sessions/{id}` | 401 | as above |
| `DELETE /auth/sessions/{id}` | 400 | `"Admin token cannot revoke sessions"` |
| Every other API route | 401 | `"Missing or invalid API token"` (unchanged) |

### Unchanged

`POST/PATCH/POST(/confirm,/cancel,/settle) /cash-flow-items*`, every
ledger / dashboard / report / reimbursement / credit / installment /
subscription / AI / notification / attachment / push / search /
reconciliation / audit / export / health endpoint keeps its v1.1.7
behaviour.

### Auth

Wire format unchanged: `Authorization: Bearer <token>`. The token can
be:

- the env admin token (legacy), OR
- a session token issued by `POST /auth/apple`.

The middleware tries admin first (constant-time compare) and falls
back to a hashed-session lookup.

### 2026-05-29 P2 偏离：audience 改为手动校验

- 变更内容：`verify_apple_identity_token` 不再把 `audience=list(...)` 传给
  `jwt.decode`，而是用 `options={"verify_aud": False}` 跳过库内 audience
  校验，再用 `aud in expected_audiences` 手动比对。
- 变更原因：实际安装的 `python-jose` 3.5 的 `jwt.decode` 只接受单字符串
  audience，传 list 会抛 `audience must be a string or None`，与 plan §P2
  草稿（PyJWT 风格的 list audience）不兼容。签名/issuer/exp 仍由 jose 验证。
- 影响范围：Phase P2（仅实现细节，对外行为不变：错误的 audience 仍拒绝）。
